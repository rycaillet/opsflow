# OpsFlow UI Plan

## Design Goal

OpsFlow should look like a polished internal operations platform used by a modern company. The interface should feel clean, organized, trustworthy, and professional.

## Core Screens

### Public Pages

- Landing Page
- Login
- Register

### Authenticated Pages

- Employee Dashboard
- Staff Dashboard
- Manager Dashboard
- Create Request
- Request Details
- Analytics
- Settings

## Layout

Authenticated pages will use a dashboard layout:

- Left sidebar navigation
- Top header
- Main content area
- Responsive layout for smaller screens

## Navigation

### Employee

- Dashboard
- My Requests
- New Request
- Settings

### Staff

- Dashboard
- All Requests
- Assigned to Me
- High Priority
- Settings

### Manager/Admin

- Dashboard
- Requests
- Analytics
- Team
- Settings

## Visual Style

### Brand Feel

- Modern SaaS
- Clean dashboard
- Professional but not boring
- Recruiter-friendly
- Not overly playful

### Suggested Colors

- Dark navy or charcoal background accents
- White/light gray content areas
- Blue primary action color
- Green success status
- Yellow warning priority
- Red urgent priority

### Components

- Buttons
- Cards
- Badges
- Tables
- Forms
- Modals
- Status labels
- Sidebar links
- Empty states
- Loading states
- Error messages

## Request Statuses

- Open
- In Progress
- Waiting
- Resolved
- Closed

## Request Priorities

- Low
- Medium
- High
- Urgent
